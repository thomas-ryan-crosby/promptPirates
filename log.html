Voyage I

The maiden voyage. Three captains, one elevator, several Costco runs, and a lot of tokens overboard.

⚓ Voyage I · Wed, June 10 2026 · 1h 03m · Aboard: Thomas Crosby, Ryan Lavin, The Rancher · Spotted off the bow: Alex Schreiner, Josh

Captain's summary

The crew compared rigging: who's running Claude vs Codex vs Fable, subscription vs API, harnesses vs raw CLI. The throughline — everyone is independently converging on the same play: build hyper-custom software for a narrow niche, then scale it. ChromeLabs does it for small businesses, Ryan does it for telehealth, The Rancher does it for a cattle operation. The conversation drifted from end-to-end testing into data moats, SEO, cattle collars, and a Dyson storm. Standard.

📌 Takeaways banked

1
Speak, don't type. Dictation (Whisper Flow, mapped to one key) runs 150–180 wpm vs 60 typing — and lets you dump context you'd never bother typing. Bonus: saved Ryan from carpal tunnel. Try it before next voyage.
2
Ship HTML, not Markdown. The crew is done with Markdown deliverables. Tell the agent "make this HTML" — system diagrams, visuals, a real consumption experience. "I want to be able to read it."
3
Meet users where they already are. The Rancher's whole app lives in Telegram — ranch hands text casual updates ("moved her, gave 3.5mg of X"), an agent parses it into the DB. They barely touch the actual UI. The interface is the chat.
4
Coding is mostly end-to-end testing now. Make everything an API so an agent can run every edge case for hours. The bugs that escape are the weird sequences (sign up → don't set password → leave → return → locked out). Describe adversarial scenarios; let it rip.
5
Fable thinks adversarially. Reported as a genuine step-change for stress-testing: spins up ~20 agents in parallel, finishes overnight test suites in ~1h that used to take 7. It actually tries to break your system. (Goes API-only after June 22.)
6
The boring stack wins. ChromeLabs' default: Firebase (auth + DB, free to absurd scale) · Vercel (hosting, auto-deploy on push, manages branches/previews) · Resend (email). Hundreds of projects for ~$20/mo. Turn a custom app around in 24–48h, show the client a clickable demo, hook them.
7
JSON files don't scale. Appending to JSON gets slower and slower as data grows. The fix is boring: move to SQL (or NoSQL/Firebase/Neon). You don't need to understand databases anymore — the agent sets it up — you just need to not store everything in flat files.
8
Harnesses are short-term leverage. Custom Claude↔Codex harnesses get out-competed as the base models improve (slash-goal, dynamic workflows make old rigs redundant). Build them to win now, don't marry them.
9
The open question: does data have a moat? Litmus test floated — does Fable outperform Harvey (law AI)? If yes, data is no moat and the SaaS is commoditized. If no, those companies are more defensible than Wall Street thinks. Crew consensus: moat holds short-term, gone in a decade or two.
