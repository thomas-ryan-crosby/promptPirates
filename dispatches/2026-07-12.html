Research & whitepapers

ReContext: Recursive Evidence Replay as LLM Harness for Long-Context Reasoning Advanced

Despite supporting context windows of 128K tokens or more, current LLMs systematically fail to surface and use evidence already present in the input. ReContext (submitted July 2, 2026) proposes a training-free harness-level fix: it uses the model’s own question-conditioned attention weights to score token relevance, materialises the top-scoring tokens back into text spans, and prepends those spans as a query-specific evidence pool before the final generation pass—without truncating or replacing the original context. Running on eight long-context benchmarks at 128K against Qwen3-4B, Qwen3-8B, and Llama3-8B, ReContext achieves best average rank on all three backbones. The mechanism requires no fine-tuning, no external retrieval index, and no additional model calls—making it a low-friction upgrade for any team running document analysis, RAG pipelines, or long-session coding agents where relevant evidence is present but unreliably used.

arXiv 2607.02509 · July 2, 2026

Your Agent’s Memories Are Not Its Own: Forged Reasoning Attacks on LLM Agent Memory and Defenses Advanced

Karamchandani, Nagasubramaniam, Zhu, and Wu at Penn State identify a new attack surface in memory-augmented agents: the reasoning trace store. The Forged Amplifying Rationale Memory Attack (FARMA) targets not what the agent knows but how it remembers deciding: in the injection phase, forged entries claim that prior actions have already been completed and can be safely skipped; in the amplification phase, self-referential entries reinforce the forgery across multiple memory reads, defeating consensus-based defenses that rely on majority-agreement across stored entries. Evasive language in the forged traces bypasses keyword-based filters. The attack succeeds against current open-weight and proprietary agent deployments with persistent memory and no output-side rationale validation. The authors introduce SENTINEL, a layered pipeline that detects forged reasoning entries through provenance tracing and cross-entry consistency checking. For any crew running Claude Code with memory plugins or custom RAG-augmented agents, this paper maps a threat class that existing injection defenses do not cover.

Articles & deep reads

Harness Engineering for Self-Improvement Advanced

Lilian Weng (co-founder, Thinking Machines Lab; former Head of Safety Research, OpenAI) published a long-form survey on July 4 arguing that recursive self-improvement will not begin with weight updates—it begins with the harness. The scaffolding around a model determines how it plans, what tools it calls, what it stores, and how its outputs are judged, and those design choices are now the lever that matters most. Weng traces the optimization frontier moving through five layers: prompt, structured context, workflow, harness code, and eventually optimizer code that rewrites the harness itself. Synthesising 35 papers, she maps seven unsolved bottlenecks: fuzzy evaluators (research taste and novelty are far harder to measure than a passing test suite), memory lifecycle management, negative-result handling, diversity collapse under self-generated training data, reward hacking, long-term compute cost, and the still-essential human role. For practitioners building production agents, the practical implication is that investments in eval quality, memory architecture, and harness observability compound in ways that frontier model upgrades alone do not.

Releases & tooling

GPT-5.6 Sol, Terra, and Luna: three-tier model family with explicit cache breakpoints goes public Intermediate

After a two-week restricted preview to government-vetted partners, OpenAI made GPT-5.6 generally available on July 9 across the API, ChatGPT, and Codex. The family introduces a durable three-tier naming convention: Sol (frontier reasoning and long-horizon agentic work, $5/$30 per million input/output tokens), Terra (balanced everyday model at GPT-5.5-competitive performance for 2× lower cost, $2.50/$15), and Luna (fastest and most affordable, $1/$6). Benchmark headliner: Sol scores 88.8% on Terminal-Bench 2.1—nudging above Claude Mythos 5’s 88.0%—and 64.6% on SWE-bench Pro. The more immediately actionable change for pipelines is the new caching model: explicit cache breakpoints let teams pin cache boundaries to specific positions in the prompt, a 30-minute minimum cache life guarantees reuse across short-interval agent calls, and cache writes are billed at 1.25× the uncached input rate (reads retain the 90% discount). Teams running heavy prompt-caching pipelines should audit cache-write frequency before migrating to GPT-5.6, as the write surcharge changes the breakeven calculation versus flat uncached pricing.

ChatGPT Work: OpenAI’s long-horizon autonomous agent ships alongside the unified desktop app Intermediate

Launched simultaneously with GPT-5.6 on July 9, ChatGPT Work is OpenAI’s answer to long-running agentic workflows: give it a goal, connect your apps, and it gathers context across files, Slack, Teams, and other integrations to return finished deliverables—documents, spreadsheets, presentations, and web apps—without continuous prompting. Scheduled Tasks extend the model’s reach beyond active sessions, letting Work pick up new signals and update project materials while the user is offline. The broader product context matters: Work, Chat, and Codex now live inside a single unified ChatGPT desktop app for macOS and Windows. Codex gains in-app diff review, inline PR annotation, and multi-repository project support. For teams already running Claude Code or similar agent harnesses, ChatGPT Work represents a direct product-layer competitor—backed by Sol’s inference budget and tight integration with Microsoft 365 and Google Workspace connectors. Available on all plans on desktop immediately, rolling out to web and mobile for paid subscribers over the following days.

Claude Code v2.1.202–207: auto mode default on cloud platforms, Opus 4.8 default, shell-injection fix Intermediate

Six releases across six days (July 6–11) with two changes that immediately affect operators running Claude Code on managed cloud infrastructure. First, auto mode is now the default on Bedrock, Vertex AI, and Foundry without requiring the CLAUDE_CODE_ENABLE_AUTO_MODE environment variable; teams that want to preserve the previous behaviour must set disableAutoMode: true in their settings. Second, the default model on all three platforms changed to Claude Opus 4.8. Other highlights across the week: v2.1.202 adds a “dynamic workflow size” setting for controlling agent concurrency and introduces OpenTelemetry attributes (workflow.run_id, workflow.name) for tracing workflow executions; v2.1.203 patches a 15–20 second macOS session-attach stall regression introduced in v2.1.196 and substantially improves background agent reliability; v2.1.205 adds an auto mode rule that blocks tampering with session transcript files; v2.1.206 adds /doctor (a check that proposes trimming bloated checked-in CLAUDE.md files) and /cd path-completion suggestions; v2.1.207 closes a shell-injection hole by rejecting ${user_config.*} variable expansion in shell-form plugin hook commands. Harnesses using plugin hooks should update to v2.1.207 immediately.
