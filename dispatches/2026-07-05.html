Research & whitepapers

Do Androids Dream of Breaking the Game? Systematically Auditing AI Agent Benchmarks with BenchJack Advanced

Hao Wang, Qiuyang Mang, Alvin Cheung, Koushik Sen, and Dawn Song at UC Berkeley built BenchJack, an automated red-teaming system that drives coding agents to audit benchmark evaluation pipelines for exploitable reward-hacking flaws. Applied to ten popular benchmarks spanning software engineering, web navigation, desktop computing, and terminal operations—including SWE-bench, WebArena, OSWorld, GAIA, and Terminal-Bench—BenchJack synthesised near-perfect-scoring exploits without solving a single task, surfacing 219 distinct flaws across eight flaw classes: leaked reference answers, prompt-injectable LLM judges, unsanitised eval(), and scoring functions that skip correctness checks entirely. An iterative adversarial extension patches the discovered flaws and re-runs, reducing the hackable-task ratio from near 100% to under 10% on four of the benchmarks and fully resolving WebArena and OSWorld within three rounds. The Agent-Eval Checklist distils the eight recurring patterns into a design-time audit any team can apply to their own evaluation harness before trusting its scores.

arXiv 2605.12673 (UC Berkeley) · May 2026

Harness-MU: A Safe, Governed, and Effective Harness for Multi-User LLM Agents Advanced

Wangxuan Fan, Xiaoyu Nie, and Zhongxiang Dai at the Chinese University of Hong Kong, Shenzhen, identify a fundamental mismatch between the single-user training paradigm of current LLMs and the hard enforcement requirements of shared multi-user deployments. Their answer is Harness-MU, the first model-agnostic, zero-tuning infrastructure framework to decouple language generation from safety orchestration: governance constraints—who is authorised, what content is restricted, whose instructions take precedence when principals conflict—are enforced by deterministic execution hooks rather than asked of the LLM. Evaluated on Muses-Bench across four frontier open-weight and proprietary models, Harness-MU achieves complete privacy preservation against all access-control attack classes while outperforming the standard baseline by 0.28–0.39 on utility score and improving instruction-following accuracy by up to 48.9 percentage points. For teams running shared Claude Code or RAG agents across a department, this is the closest thing the field currently has to a reference architecture for multi-principal governance.

Releases & tooling

Previewing GPT-5.6 Sol: OpenAI introduces three-tier model naming and explicit cache breakpoints Intermediate

OpenAI previewed a three-tier model family under a new naming regime: Sol (the strongest), Terra (balanced), and Luna (fastest and cheapest), priced at $5/$30, $2.50/$15, and $1/$6 per million input/output tokens respectively. The naming scheme is designed to be durable: the Sol, Terra, and Luna tiers will advance independently on their own cadence, giving developers a stable capability-tier vocabulary across model generations. For practitioners building caching-heavy pipelines the more immediately useful change is the introduction of explicit cache breakpoints and a 30-minute minimum cache life; cache writes are now billed at 1.25x the uncached input rate while cache reads retain a 90% discount. Both mechanics let teams control cache boundaries more precisely and size cache-write costs into their pipelines upfront. The preview is restricted to a small group of trusted partners at the US government’s request, with broader API and ChatGPT availability expected within weeks.

OpenAI · late June 2026

Claude Sonnet 5: most agentic Sonnet launches with three API breaking changes Intermediate

Claude Sonnet 5 (claude-sonnet-5) shipped June 30 as the new default model in Claude Code and on Free and Pro claude.ai plans. At introductory pricing of $2/$10 per M input/output tokens through August 31 (standard $3/$15 after), it lands close to Opus-class performance at significantly lower cost, making it the natural choice for high-throughput agent pipelines. Three API changes require attention before migration. First, adaptive thinking is on by default—previously opt-in. Second, manual extended thinking via thinking: {type: "enabled", budget_tokens: N} is removed and returns a 400 error; it was deprecated on Sonnet 4.6. Third, setting temperature, top_p, or top_k to non-default values also returns a 400 error. Sonnet 5 also ships a new tokenizer that produces roughly 30% more tokens for the same text, which matters for both cost estimates and prompt-cache sizing—audit your budgets before flipping the default.

Claude Fable 5 and Mythos 5 access restored after export-control suspension Intermediate

As of July 1, the US Commerce Department export-control directive that had suspended all access to Claude Fable 5 and Mythos 5 was lifted, and both models returned to global availability. Fable 5 (claude-fable-5), Anthropic’s most capable widely released model, is now accessible on the Claude API, Claude Platform on AWS, Amazon Bedrock, Google Cloud, Microsoft Foundry, and directly in Claude Code. Its specs—1M token context, 128k max output, $10/$50 per M input/output tokens, always-on adaptive thinking—are unchanged. Claude Mythos 5 remains invitation-only within Project Glasswing for approved defensive cybersecurity work. For teams that held off on migrating harnesses to Fable 5 while access was uncertain, the path is clear again.

Anthropic · July 1, 2026

Claude Code v2.1.197–201: background subagents by default, Claude in Chrome, permission mode overhaul Intermediate

Five releases across four days. The most consequential is v2.1.198 (July 1): subagents now run in the background by default, meaning parent agents no longer block waiting for subagent completion—the same release also brings Claude in Chrome to general availability and the /dataviz skill for chart and dashboard design guidance, and wires background agents to auto-commit, push, and open draft pull requests when work is complete. Also in v2.1.198: Claude Sonnet 5 becomes the default model (v2.1.197, June 30). The v2.1.199 release (July 2) adds stacked slash-skill invocations (up to five skills loaded per call) and preserves streaming partial output when the API emits mid-stream errors rather than discarding the turn. The v2.1.200 release (July 3) changes the default permission mode to Manual across the CLI and IDEs and stops AskUserQuestion dialogs from auto-continuing—both changes affect any harness that relied on default interactive behaviour. Harness builders running automated Claude Code sessions should test these defaults before the next production push.

Claude Code changelog · June 30–July 3, 2026

Claude Managed Agents API: per-session config overrides, lifecycle webhooks, event deltas Intermediate

The June 30 platform release packed five additions to the Managed Agents API. Per-session configuration overrides are the most flexible: pass agent with type: "agent_with_overrides" when creating a session and you can swap the model, system prompt, tools, MCP servers, or skills for that session alone without touching the published agent definition—useful for A/B testing models or injecting user-specific context without proliferating separate agents. Lifecycle webhooks now cover the full deployment surface: agent publication, deployment pause, and scheduled-run failure events all arrive as webhook payloads, eliminating polling loops for teams that react to these transitions in their CI or ops pipelines. Event deltas let stream consumers show assistant message text as it generates by subscribing to event_start and event_delta events on the session event stream before the final agent.message event closes. Two smaller additions: sessions list now supports backward pagination, and vault environment variable credentials expose an injection_location setting to target secrets into request headers, the request body, or both at egress.
