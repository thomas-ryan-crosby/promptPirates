Research & whitepapers

SPOQ: Specialist Orchestrated Queuing for Multi-Agent Software Engineering Advanced

Proposes a three-part coordination method for multi-agent coding pipelines: wave-based topological dispatch that computes parallel execution waves from task dependency graphs, dual validation gates applied both before and after each wave to cut rework cycles, and a Human-as-an-Agent (HaaA) slot so a human specialist can join decomposition or be consulted mid-run without stopping the queue. Includes a longitudinal deployment study and open-weights replication. Relevant to anyone building orchestration layers on top of Claude Code or similar agent harnesses.

arXiv 2606.03115 (Carbowitz et al.) · June 2, 2026

Articles & deep reads

Project Glasswing: An initial update Advanced

Anthropic’s first public accounting of what Claude Mythos Preview has found in the wild: more than 10,000 high- and critical-severity vulnerabilities across 50-plus partner organizations, with 6,202 high/critical findings surfaced by scanning 1,000 open-source projects (23,019 total bugs identified). Notable examples include a certificate-forgery exploit in wolfSSL and a 17-year-old FreeBSD remote code execution flaw (CVE-2026-4747) discovered and exploited fully autonomously. The sobering coda: open-source maintainers are now the bottleneck, with vulnerability discovery outpacing the capacity to patch. Program since expanded to 150 more organizations in 15-plus countries.

Anthropic research blog · May 26, 2026

Prompting Claude Fable 5: behavioral differences and scaffolding changes Advanced

Anthropic’s practitioner guide to what actually changed at the harness level with Fable 5. Key shifts: individual requests can run for many minutes at high effort—adjust client timeouts before migrating. The model dispatches parallel sub-agents more readily than Opus 4.8, so orchestrators need asynchronous patterns rather than blocking. Prompts that tell the model to echo or reproduce its internal reasoning now trigger the reasoning_extraction refusal category; use structured thinking blocks instead. Includes ready-to-paste system-prompt snippets for grounding progress claims, scoping autonomous action, and managing a persistent memory file.

Anthropic platform docs · June 9, 2026

Releases & tooling

Claude Fable 5 and Claude Mythos 5 Advanced

Anthropic’s new flagship tier lands June 9: Claude Fable 5 (claude-fable-5) is generally available on the Claude API, AWS Bedrock, Vertex AI, and Microsoft Foundry; Claude Mythos 5 (claude-mythos-5) is restricted to Project Glasswing partners and carries no safety classifiers. Both share a 1M-token context window, 128k max output, and $10/$50 per million input/output tokens. Adaptive thinking is always on—thinking: {type: "disabled"} is rejected. New API surface: stop_reason: "refusal" returns as HTTP 200 with a stop_details.category field (cyber, bio, frontier_llm, reasoning_extraction); a server-side fallback beta (server-side-fallback-2026-06-01) lets you name fallback models on the request and have the API handle retries automatically. Note also: Claude Sonnet 4 and Claude Opus 4 retire June 15, 2026.

Anthropic API docs · June 9, 2026

Claude Code v2.1.172: sub-agents can spawn sub-agents Intermediate

The headline addition in this week’s Claude Code release is nested agent trees: sub-agents can now themselves spawn sub-agents, up to five levels deep. Alongside that, a search bar lands in the /plugin marketplace browser, Amazon Bedrock now auto-reads AWS region from ~/.aws config files when AWS_REGION is unset, and a OTEL metric (claude_code.lines_of_code.count) gains a model attribute. Bug fixes include permanently stuck 1M-context sessions and a repeating “image could not be processed” error. Pairs directly with Fable 5’s improved sub-agent delegation capabilities.

Claude Code changelog · June 10, 2026

Microsoft Build 2026: seven MAI models, including MAI-Thinking-1 and MAI-Code-1-Flash Intermediate

At Build 2026 (June 2), Microsoft shipped seven in-house MAI models, with two immediately relevant to the AI coding agent stack. MAI-Thinking-1 is Microsoft’s first reasoning model built entirely on commercially licensed enterprise data with no distillation from third-party models—the independence claim matters for enterprise compliance. MAI-Code-1-Flash is the coding model now powering GitHub Copilot across Free, Student, Pro, Pro+, and Max plans, selectable from the VS Code model picker. The suite also includes MAI-Transcribe-1.5 (SOTA transcription across 43 languages, 5x faster than competing models), MAI-Voice-2, and MAI-Image-2.5.

Microsoft AI · June 2, 2026

GitHub Copilot App and SDK: an agent-native desktop, now generally available Intermediate

Also out of Build 2026: GitHub shipped the Copilot App, a desktop client built around a single “My Work” view that surfaces active sessions, issues, pull requests, and background automations across connected repositories. The Copilot SDK reached general availability at the same time, giving developers a stable API to embed Copilot’s agentic engine in their own tools and services. The CLI companion got a significant refresh—rubber duck mode, prompt scheduling, and voice input all moved from experimental to GA. Worth tracking as the reference architecture for how Microsoft envisions agent-first developer tooling.
