Research & whitepapers

Agentic Very Much! Adoption of Coding Agent in New GitHub Projects Intermediate

A follow-up empirical study by Romain Robbes and co-authors on how quickly coding agents are taking over new GitHub projects. The headline: adoption is more than twice as high in the new cohort compared to the previous study, and the proportion of AI-assisted commits is meaningfully higher, even though the researchers note current detection methods likely undercount it. At the time of writing, Claude Code alone accounts for roughly 4.5% of all public repository commits on GitHub—up from 100,000 commits per week in September 2025 to 2.6 million now. Concrete baseline for anyone making the case internally that agentic coding is already the norm, not a future possibility.

arXiv 2606.07448 (Robbes et al.) · June 5, 2026

Human oversight of agentic systems in practice: Examining the oversight work, challenges, and heuristics of developers using software agents Advanced

Shipi Dhanorkar, Samir Passi, and Mihaela Vorvoreanu interviewed 17 experienced developers to map what oversight actually looks like in the wild, bridging the gap between normative frameworks and lived practice. They identify four forms of emergent oversight work: a priori control (shaping the agent before it runs), co-planning (negotiating the task in real time), real-time monitoring (watching execution), and post hoc review (auditing after the fact). The study also documents the heuristics developers have improvised to manage failure modes the agents themselves were not designed to surface. Required reading for anyone formalizing a human-in-the-loop workflow or writing guidance for a team deploying long-horizon agents.

arXiv 2606.05391 (Dhanorkar, Passi, Vorvoreanu) · June 3, 2026

Articles & deep reads

What we learned mapping a year’s worth of AI-enabled cyber threats Advanced

Anthropic’s threat-intelligence team analyzed 832 accounts banned for malicious cyber activity between March 2025 and March 2026, documenting 13,873 observed actions across 482 unique MITRE ATT&CK techniques covering all 14 tactics. The two findings that matter most to practitioners: the share of medium-to-high-risk actors using AI jumped from 33% to 56% over the study period, and the pattern of AI use shifted inward—away from initial access (AI-assisted phishing fell) and toward post-compromise activity inside networks (account discovery rose). Attackers are pushing AI deeper into the kill chain, not just using it for lure generation. Worth reading alongside any security posture review for agentic systems you’re deploying.

Anthropic research blog · June 3, 2026

Refusals and fallback: integrating Claude Fable 5’s new stop reason Advanced

The practitioner-level guide to the most disruptive API change Fable 5 ships: classifier refusals that return HTTP 200 with stop_reason: "refusal" and a stop_details.category field—currently one of cyber, bio, frontier_llm, or reasoning_extraction. Partial output mid-stream can also end in a refusal, so the content must be discarded. The guide covers all three retry paths: server-side fallback (pass fallbacks with the server-side-fallback-2026-06-01 beta header, API handles the hop internally), SDK middleware (TypeScript, Python, Go, Java, C#), and manual retry with fallback credit to avoid paying the prompt-cache cost twice. Sticky routing means a conversation that fell back stays on the fallback model for roughly an hour—account for this in your monitoring. If you ship anything on Fable 5, this page should be open in a browser tab during the build.

Anthropic platform docs · June 9, 2026

Releases & tooling

June 15 double-header: Agent SDK billing split and Claude Sonnet 4 / Opus 4 retirement Intermediate

Two Anthropic deadlines land tomorrow and require action before end of day. First, the billing split: starting June 15, usage via the Claude Agent SDK, claude -p, Claude Code GitHub Actions, and third-party apps built on the SDK moves off your subscription’s interactive limit onto a separate monthly credit metered at full API list prices—$20 on Pro, $100 on Max 5x, $200 on Max 20x, non-pooled and non-rollover. Interactive Claude Code in the terminal is unaffected. Second, the model retirement: claude-sonnet-4-20250514 and claude-opus-4-20250514 are retired today; any integration still calling either will receive errors. Migrate to claude-sonnet-4-6 and claude-opus-4-8 respectively. Separately, a softer but important API change: temperature, top_p, and top_k are deprecated on Claude Opus 4.7 and later—setting them to non-default values returns a 400 error.

Anthropic API docs · June 15, 2026 (effective)

Claude Code v2.1.175–176: enforceAvailableModels and usage attribution Intermediate

The pair of releases this week rounds out the enterprise model-governance story opened by nested sub-agents in v2.1.172. The headline addition is enforceAvailableModels (v2.1.175), a managed setting that constrains which Claude model the default picker may resolve to; user or project settings can no longer widen a managed allowlist, giving platform administrators hard control over which models reach production. Alongside that, v2.1.174 adds usage attribution to the VSCode Account dialog—cache misses, long-context usage, and subagent calls are now broken out separately, making cost analysis against those workloads tractable. Bedrock credential caching is also extended to honor the actual expiration rather than capping at one hour.

Claude Code changelog · June 11–12, 2026

Gemini CLI retires June 18: transition to Antigravity CLI Intermediate

Google’s open-source, Apache 2.0 terminal coding agent stops serving Google AI Pro and Ultra subscribers on June 18, folded into the broader Antigravity platform announced at Google I/O. Antigravity CLI is not open source; it is built in Go (faster execution, native async), shares the same agent harness as the Antigravity 2.0 desktop application, and carries over familiar primitives—agent skills, hooks, and subagents reappear as Antigravity plugins. Consumer access ends in four days; enterprise access on Gemini Code Assist is unchanged for now. If your team runs any Gemini CLI–based automation, the migration guide at the Google Developers Blog is the authoritative starting point.

Google Developers Blog · May 19, 2026 (effective June 18)

WebMCP: a browser-native tool protocol for AI agents, now in Chrome origin trial Intermediate

Announced at Google I/O 2026 and co-developed with Microsoft, WebMCP is a proposed W3C standard that lets websites declare named JavaScript functions and HTML forms as machine-readable tool endpoints—"search inventory," "initiate checkout," "submit support request"—which browser-based AI agents invoke with explicit user authorization rather than by scraping screen layouts. The Chrome 149 origin trial is shipping this month; Firefox has committed to Q3 support, Safari to Q4. Early data from the proponents cites 67% fewer task errors compared to visual scraping. Whether you are building web agents or hardening web applications against them, understanding what the browser can now expose to an agent natively is no longer optional.
