Tools, testing & the boring stack

The maiden voyage of the Prompt Pirates

Date
June 10, 2026 · 1h 03m
Aboard
Ryan Lavin, Thomas “Ryan” Crosby, Nathan Allen
Hailed
Alex Boudreaux, Josh Meister, Vincenzo Caronna

Three operators compared rigging — Claude vs. Codex vs. Fable, subscriptions vs. API, custom harnesses vs. raw CLI — and found they'd independently converged on the same play: build hyper-custom software for one narrow niche, then scale it. The conversation ran from end-to-end testing through data moats, SEO, and cattle collars. What follows is the discussion in brief, then the treasure.

1Tooling & the subsidy era

The crew is running every tool at once and burning through usage limits in hours. The working consensus: ride the venture-subsidized "cheap tokens" era for as long as it lasts, because usage-based billing is where everything is heading (Fable goes API-only after June 22). Custom harnesses that bolt Claude's planning onto Codex's engineering are losing their edge as the base models absorb those tricks — slash-goal and dynamic workflows already make older rigs redundant. The lesson underneath: harnesses are short-term leverage; build them to win now, don't marry them.

2The interface is the chat

Nathan's ranch-manager app is the standout example. It runs almost entirely through Telegram: ranch hands text casual updates — "moved her to the back pasture, gave 3.5mg of X" — and an agent parses the natural language into structured records. The users barely open the actual app UI; they just ask the group chat "when did this cow last get treated?" and get plain-English answers from an LLM with privileged database access.

The takeaway the table kept circling: meet users where they already are. A chat box they already live in beats a polished dashboard nobody opens.

3Coding has become adversarial testing

Ryan's days are mostly end-to-end testing, not writing features. The architectural shift driving this: make everything an API, so an agent can drive every edge case for hours. The bugs that escape are never the happy path — they're weird sequences (sign up → don't set a password → leave → come back → locked out). You catch those by describing adversarial scenarios and letting the agent run them overnight.

Fable drew specific praise here as a genuine step-change: it "thinks adversarially," spins up ~20 agents in parallel, and finishes suites in about an hour that used to take seven. It actively tries to break the system rather than confirming it works.

"If it runs, it runs. It doesn't write code like a human — it's totally alien to me."

4The boring stack wins

Thomas's ChromeLabs turns around custom small-business software in 24–48 hours on a deliberately boring default stack: Firebase (auth + database, free to absurd scale), Vercel (hosting, auto-deploy on git push, painless branch previews), and Resend for email. Hundreds of projects run for roughly $20/month. The go-to-market is just as lean: build a clickable demo, hand it over free to play with, then price it — "where do I sign?" beats any pitch deck.

A cautionary counterpoint surfaced too: flat JSON files don't scale — lookups crawl as data grows. The fix is unglamorous (move to SQL / a real DB), and the agent will set it up for you; you just have to stop storing everything in flat files.

5Moats, SEO & the long view

The open debate of the night: does data still have a moat? The litmus test floated was Harvey, the law AI — if a general model like Fable starts outperforming it, then proprietary data is no moat and the vertical SaaS gets commoditized; if not, those companies are more defensible than the market thinks. Crew consensus: the moat holds in the short term, but breaks down over a decade or two.

On SEO, skepticism toward the traditional "optimization" industry, but real interest in its successor — getting your name mentioned enough across the web that it lands in the next model's pre-training set. Nathan reported going from non-existent to #1 on both Google and AI search for "Mississippi Red Angus" in about two weeks.

⚓ The Treasure

Key lessons from Voyage I

  • Meet users in the chat they already live in. A Telegram agent that parses casual messages into structured data beats a dashboard nobody opens.
  • Make everything an API, then test adversarially. The escaped bugs are weird sequences — describe them as scenarios and let agents (Fable especially) run them in parallel overnight.
  • Ship the boring stack. Firebase + Vercel + Resend turns a custom app around in 24–48h for ~$20/mo. Demo free, then price. And don't store real data in flat JSON.
  • Niche first, then scale. Everyone serious is converging on hyper-custom software for one vertical, then templatizing it.
  • Harnesses are temporary; base models catch up. Use them to win today, but don't build your identity around a rig the next model release will absorb.
